# PIC LED Blink — Timer0 Interrupt Demo

A bare-metal embedded C project for **PIC microcontrollers** (targeting the eCee development board) that blinks an LED using the **Timer0 overflow interrupt** — no busy-wait loops.

## Overview

This project demonstrates a fundamental embedded programming pattern:

- **Timer0** is configured in 8-bit mode with its internal clock source and no prescaler.
- On every Timer0 overflow interrupt, a counter is incremented.
- Once the counter reaches **20,000**, LED1 (connected to **RB0**) is toggled and the counter resets.
- The main loop is intentionally empty — all logic lives in the ISR.

## How It Works

```
Timer0 overflow → ISR fires → increment counter
                                    │
                              count == 20000?
                               ├── Yes → toggle LED1, reset count
                               └── No  → clear TMR0IF, return
```

**Timer reload:** On each interrupt, `TMR0` is incremented by 8 (`TMR0 = TMR0 + 8`) to compensate for interrupt latency and keep timing accurate.

**LED toggle period** depends on `Fosc` and the Timer0 reload value. With a 20 MHz clock and no prescaler, each Timer0 overflow occurs every ~51.2 µs, making the LED period approximately:

```
20,000 overflows × 51.2 µs ≈ 1.024 s  →  ~0.5 Hz blink
```

## Project Structure

```
.
├── main.c          # Entry point — peripheral init and infinite loop

├── timer0.h        # Timer0 public API
```
